# 🔄 Паттерны получения и передачи данных - Сравнение

## Быстрое сравнение

| Паттерн | Направление | Латентность | Сложность | Когда использовать |
|---------|-------------|-------------|-----------|-------------------|
| WebSocket | ↔️ Двусторонняя | ⚡ Очень низкая | 🔴 Средняя | Real-time чат, биржи |
| REST Polling | → Односторонняя | 🐌 Высокая | 🟢 Низкая | Редкие обновления |
| SSE | → Server→Client | ⚡ Низкая | 🟡 Низкая | Уведомления, логи |
| Long Polling | → Server→Client | 🟡 Средняя | 🟡 Средняя | Fallback для WebSocket |
| gRPC | ↔️ Двусторонняя | ⚡ Очень низкая | 🔴 Высокая | Микросервисы |
| Message Queue | → Асинхронная | 🟡 Средняя | 🔴 Высокая | Event-driven системы |
| GraphQL Sub | ↔️ Двусторонняя | ⚡ Низкая | 🔴 Средняя | GraphQL приложения |
| Webhooks | → Push модель | 🟡 Средняя | 🟢 Низкая | Payment callbacks |
| WebRTC | ↔️ P2P | ⚡ Минимальная | 🔴 Очень высокая | Video/audio streaming |

## Матрица выбора

Нужен real-time?
├─ Да → Двусторонняя связь?
│ ├─ Да → WebSocket / gRPC
│ └─ Нет → SSE
└─ Нет → Простота важнее?
├─ Да → REST Polling
└─ Нет → Message Queue
